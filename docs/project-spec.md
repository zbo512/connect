## 0. v11 변경 요약

v10 대비 변경된 사항:

- **s-mod-lib 제거** — 별도 화면 폐기, 빌더 모듈 설정 탭으로 완전 통합
- **s-builder 모듈 목록 확장** — 기존 8개 → 전체 16개 모듈 (SNS링크·보장분석·청구대행·매거진·유튜브·인스타·소식구독·후기·FAQ 추가)
- **활성/비활성 상태 텍스트 전환** — 활성: 현재 데이터, 비활성: 모듈 설명 (data-on/data-off)

- **s-notify 전면 재설계** — 날짜 그룹·카드형·읽음 카운터·모듈 추천 알림
- **s-customer 이모지 → SVG 전체 교체**
- **고객 페이지 케이스 이름 변경** — C1~C4 제거 → 미니멀형·표준형·콘텐츠형·이벤트형
- **Identity 타입 체계 정의** — 케이스별 타입명·구성 확정 / 표준형(c2) 사진·카드 융합형 상세 스펙 추가

## 17. s-notify 디자인 스펙 (v11 신규)

### 구조

- Nav: "알림" + 우측 읽음 카운터 (N/M 읽음 → 모두 읽음)
- 배경: gray-100
- 레이아웃: 카드형 (gap 8px, border-radius 16px)

### 날짜 그룹

`오늘` / `어제` / `이번 주` — 섹션 레이블 (11px, gray-400, uppercase)

### 읽음/안읽음 상태

| 상태 | 카드 배경 | 아이콘 배경 | SVG 컬러 | 우측 표시 |
|------|----------|------------|---------|---------|
| 안읽음 | blue-50 | transparent | #3182F6 | — |
| 읽음 | white | transparent | #8B95A1 | ✓ 체크 (::after) |

### 알림 타입

| 타입 | 예시 | 탭 동작 |
|------|------|---------|
| 고객 활동 | 상담 예약·보험 조회·페이지 방문 | — |
| 시스템 경고 | 이벤트 잔액 부족·HR 동기화 실패 | 해당 에러 화면 이동 |
| 정보·공지 | 주간 방문 리포트 | — |
| 모듈 추천 | 보험 가이드·데일리 뉴스 권장 | 인라인 [켜기] 버튼 → 빌더 토글 즉시 ON |

### 읽음 카운터 동작

- 진입 시: `N / M 읽음` (파랑, 클릭 가능)
- 알림 탭마다 N 증가, 실시간 갱신
- 모두 읽으면: `모두 읽음` (회색, 비활성)

### Identity 타입 (v11 신규)

| 케이스 | 타입명 | 구성 |
|--------|--------|------|
| 미니멀형 (c1) | 텍스트형 | 소형 아바타 + 이름 + 한줄 소개 + 카카오 버튼 |
| 표준형 (c2) | 사진·카드 융합형 | 풀블리드 사진 (320px) + 자격증 뱃지 + 하단 정보 카드 |
| 콘텐츠형 (c3) | 텍스트형 (현행 유지) | — |
| 이벤트형 (c4) | 텍스트형 (현행 유지) | — |

### 표준형 (c2) Identity 상세 스펙

**사진 존**

- `height: 320px`
- `object-fit: cover` / `object-position: top center`
- 배경 없음, 하단 페이드 없음
- 좌상단 뱃지: 흰 반투명 pill — 자격증 표시

**정보 카드**

- 이름: 20px / 700 / 중앙 정렬
- 소속: 13px / gray-500 / 중앙 정렬
- 스탯 3칸: 평점 · 상담 고객 · 경력 (border 구분)
- 소개 문구: 14px / 중앙 정렬
- CTA: 카카오 버튼 (full width)

**실제 서비스 전환 시**

- `<img src="...">` URL 교체로 적용
- `object-position`으로 크롭 조정

### 아이콘 정책

이모지 전면 금지. 전체 flat solid SVG 교체 완료.
